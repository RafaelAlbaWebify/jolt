# Application redesign acceptance criteria

- Applications loads without an endpoint error against the live database.
- Primary view is a compact stage board or equivalent pipeline, not a tall form stack.
- Opportunity inspector contains only one application handoff action.
- Application detail uses tabs or clearly separated sections for overview, tasks, interviews, contacts, documents, and timeline.
- Stage changes are concise actions with confirmations, not oversized text tiles.
- Submission, follow-up, interview, offer, and closure dates are structured.
- One real role can be moved from preparation to applied to interview to closed without disappearing or losing data.
- Visual validation passes at 1680x945.
