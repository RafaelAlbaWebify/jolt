# JOLT UX audit — 2026-07-23

Audit source: `JOLT_WORKFLOW_AUDIT_20260723_121334.zip`, viewport 1680×945.

## Technical result

- No page errors.
- No failed requests.
- No HTTP 4xx/5xx responses.
- Applications board loads in 1.155 s.
- Application workspace opens in 0.501 s.
- Opportunity inspector opens in 0.611 s.

## Human-UX findings

1. The Applications board is constrained to roughly 1130 px inside a 1680 px viewport, leaving large unused margins while squeezing five lanes.
2. Cards wrap aggressively and the Closed lane content is visibly clipped.
3. Workspace tabs are static labels rather than usable navigation.
4. The workspace opens as a shallow strip instead of a full working surface.
5. The inspector boundary check can falsely pass because the audit opens the first opportunity rather than deliberately selecting one with an application workflow.

## Acceptance corrections

- Widen Applications to use the available viewport.
- Ensure lane/card text remains readable without clipping.
- Make workspace tabs real controls with visible content panels.
- Give the workspace a practical minimum height and internal scrolling.
- Select a pursued/applied opportunity when checking inspector boundary.
- Keep `inspector_application_workflows == 0` as a release gate.
